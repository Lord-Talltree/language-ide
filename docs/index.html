L-ide: A GPS Protocol for AI Conversations

Published: November 22, 2025 | Author: Ruben Kai Baden
TL;DR: I built an open protocol that turns AI conversations into queryable, visual maps. Think "Git for chat" - it tracks what was discussed, what's being built, and catches contradictions before they become problems.

The Problem: AI Memory is Broken

If you've used ChatGPT or Claude for anything complex (writing code, planning a project, drafting a document), you've hit the wall: after 20-30 messages, the AI forgets.

Current "solutions" are inadequate:

  • Vector stores (RAG): Black boxes. You can't SEE what the AI remembers.
  • Copy-pasting context: Manual, error-prone, breaks flow.
  • Cloud memory services: Privacy risks, vendor lock-in, opaque.

What we need is a persistent, queryable, visual map of the conversation - for both humans and AI.

The Solution: L-ide Protocol

L-ide is a local-first protocol that automatically builds a "Meaning Graph" from AI conversations. It consists of three core innovations:

1. The Meaning Graph Schema

Instead of storing conversations as flat text, L-ide extracts structured meaning:

{
  "nodes": [
    { "id": "evt_1", "type": "Event", "label": "build" },
    { "id": "goal_2", "type": "Goal", "label": "deploy" },
    { "id": "ent_3", "type": "Entity", "label": "database" }
  ],
  "edges": [
    { "source": "evt_1", "target": "ent_3", "role": "Patient" },
    { "source": "goal_2", "target": "evt_1", "role": "Sequence" }
  ]
}

Key insight: Conversations have structure (goals, events, entities, relationships). By making this explicit, we enable:

  • Navigation: "What was the original goal?"
  • Verification: "Did I contradict myself?"
  • Resumption: "What was I working on last week?"

2. The Interceptor Pattern

The core architecture is a sidecar that sits between the user and the AI:

User → [L-ide Interceptor] → AI Agent
         ↓                           ↓
   Extract Meaning            Check Consistency
         ↓                           ↓
   Update Graph ← Run Checkers ← Response

The critical innovation: The Interceptor runs BEFORE the AI responds. If it detects a problem (ambiguity, contradiction), it injects a warning:

User: "Make it blue."
Interceptor: [WARNING: Ambiguous pronoun 'it']
AI: "Which part should be blue? The button or background?"

3. The Checker Algorithms

L-ide uses pluggable checker algorithms to detect issues:

ContinuityChecker

Detects contradictions across time: "The car is red" → later → "The car is blue" = CONFLICT

AmbiguityChecker

Flags vague terms: Pronouns without clear referents ("it", "this") trigger warnings

DiscourseChecker

Extracts narrative structure: "I fell BECAUSE I tripped" → creates CAUSE edge

Why This Matters: The Three Use Cases

1. For Developers: "Stop Repeating Yourself"

Building a feature with ChatGPT. It suggests something that conflicts with Message 3. L-ide catches it:

[WARNING: Contradiction. You decided PostgreSQL, 
but this plan assumes MongoDB. Clarify database.]

2. For Writers: "Track Your World"

Novel character has blue eyes in Ch1, brown in Ch10. L-ide flags the continuity error.

3. For Teams: "Audit Your AI"

Customer service bot promises a refund, but policy forbids it. L-ide blocks the violation.

The Architecture: Protocol-First

L-ide is a protocol, not a monolithic app:

  1. Backend (FastAPI + SQLite): Processes text, builds graph, stores sessions
  2. Clients (thin wrappers): CLI, Chrome Extension, VSCode, API integration

Key principle: The backend holds ALL logic. Clients are dumb display layers.

Benefits:

  • Privacy: Runs locally. Data never leaves your machine.
  • Portability: Works with ANY AI (ChatGPT, Claude, local models)
  • Extensibility: Community can build clients

Prior Art & Novelty

What already exists:

  • Knowledge graphs (Wikidata) - static, not conversation-derived
  • Semantic parsers (AMR) - research tools, not production
  • RAG systems (LangChain) - vector-based, not graph-based

What's novel in L-ide:

  1. Real-time graph construction from conversation
  2. Interceptor architecture (warnings before AI responds)
  3. Pluggable checker system
  4. Local-first protocol

Conclusion

AI memory remains broken. L-ide solves this with a visual, queryable graph + proactive verification + local-first architecture.

This is git for conversations. Just as git made collaboration on code manageable, L-ide makes collaboration with AI reliable.
